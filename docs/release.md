# Release Checklist

Use this checklist before publishing `disciplines`.

## Candidate Gate

```sh
npm ci
npm run check:release
```

`check:release` runs:

- TypeScript typecheck
- discipline/schema validation
- resolver fixtures
- catalog and adapter discovery checks
- end-to-end CLI workflow checks
- package tarball smoke checks
- `npm publish --dry-run --access public`

Do not publish if any step fails.

## Manual Review

- Confirm `npm view disciplines version` is lower than the local `package.json` version.
- Confirm `README.md` and [cli.md](cli.md) describe every public command and source format.
- Confirm [agent-discovery.md](agent-discovery.md) matches the current Claude Code, Codex, and Cursor generated files.
- Confirm `disciplines add`, `disciplines install`, `disciplines use`, `disciplines doctor`, and `disciplines cleanup` still behave advisory-only and do not install MCPs, CLIs, or services automatically.
- Confirm `disciplines.json` and `disciplines-lock.json` are documented and their schemas are included in the package tarball.
- Confirm `dist/` is generated by `prepack`; do not commit generated build output.

## Publish

Publishing is intentionally manual:

```sh
npm publish --access public
```

After publishing, verify:

```sh
npm view disciplines version
npx disciplines --version
npx disciplines --help
npx disciplines search frontend
npx disciplines smoke
```
